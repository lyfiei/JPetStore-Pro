##   关键技术点

### 1. 跨域配置（CORS）

后端需要配置 CORS，允许前端访问：

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://localhost:5173")
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowedHeaders("*")
                .allowCredentials(true);
    }
}
```

### 2. JWT Token 认证

- 登录成功后返回 Token
- 前端存储 Token 到 localStorage
- 每次请求携带 `Authorization: Bearer <token>`
- 后端拦截器验证 Token

### 3. 统一响应格式

所有 API 返回：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### 4. 前端路由守卫

需要登录的页面添加 `requiresAuth: true`，路由守卫自动检查 Token

##  💡 开发建议

### 推荐开发顺序

1. **先完成后端基础配置**（CORS、异常处理、统一响应）
2. **开发认证模块**（登录、注册、Token）
3. **开发商品目录模块**（公开接口，优先级高）
4. **开发购物车模块**（需要登录）
5. **开发订单模块**
6. **前端逐个页面对接**

### 注意事项

1. ✅ 保留旧的 Thymeleaf Controller（新旧并存）
2. ✅ API 路径使用 `/api/v1/` 前缀
3. ✅ 前端开发时使用 Mock 数据
4. ✅ 注意跨域问题
5. ✅ Token 过期处理
6. ✅ 错误提示友好
